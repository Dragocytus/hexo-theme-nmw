var VS let

发布 : 2018-05-23 分类 : ES6 浏览 :
  • 目前，如果大量使用let，为了保证兼容性，webpack会将let转化为var语法（这就会生成冗余代码）。其实，结合ES6的模块化规范，还是推荐使用：var，以保证兼容老旧浏览器情况下减小代码体积。
  • 随着浏览器逐步升级，let才是未来主流。

ES5下的var

分别点击两段文字，控制台输出的都是2。按道理应该是1或2。这是因为：JavaScript引擎内部会记住上一轮循环的值，初始化本轮的变量i时，就在上一轮循环的基础上进行计算。也就是说，var声明的变量的作用域并不是自身作用域。
